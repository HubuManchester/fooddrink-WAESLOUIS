# mockapi.io Data Source Setup Guide

This project supports reading and adding food data from mockapi.io.

## 1. Current Data Source

The project originally used local mock data located in:

```text
FoodDrinkApp/Services/FoodCatalogService.cs
```

It has been updated to:

- Prioritise mockapi.io REST API
- Use local fallback data when the API is not configured or the network is temporarily unavailable, ensuring the app does not crash

## 2. Creating a Data Table on mockapi.io

1. Open [mockapi.io](https://mockapi.io)
2. Create or enter a Project
3. Click Add Resource
4. Recommended Resource name:

```text
foods
```

5. Add the following fields:

| Field Name | Suggested Type | Description |
|---|---|---|
| name | String | Food or drink name |
| category | String | Category, e.g. Breakfast, Lunch, Drink |
| description | String | Description |
| calories | Number | Calories |
| protein | Number | Protein |
| carbs | Number | Carbs |
| fat | Number | Fat |
| allergyNote | String | Allergy note |
| tags | String | Search tags |

The `id` field is auto-generated by mockapi.io and does not need to be added manually.

## 4. Adding the API URL to the Project

After creating a Resource, mockapi.io generates an address similar to:

```text
https://682xxxx.mockapi.io/api/v1/foods
```

Open:

```text
FoodDrinkApp/Services/MockApiConfig.cs
```

Change `EndpointUrl` to your address:

```csharp
public const string EndpointUrl = "https://682xxxx.mockapi.io/api/v1/foods";
```

After restarting the app:

- The main page list will fetch food data from mockapi.io
- The add-item page will POST to mockapi.io on save
- The detail page will fetch individual records via the API

## 5. How to Explain During the Screencast

You can say something like:

> The project initially used local mock data. To better reflect a real mobile app data source, I migrated the data layer to the mockapi.io REST API. `FoodCatalogService` uses `HttpClient` to fetch the food list, add new records, and retrieve details. If the network is unavailable, the app uses local fallback data to avoid crashing during the demo.
